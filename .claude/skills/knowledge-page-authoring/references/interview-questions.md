# Interview Question Bank

**Example to match:** `interview-prep/EA_Senior_Interview_Questions.md`

## When to use
A collection of interview questions for a specific role/seniority level,
meant to help a candidate prepare — not a general knowledge quiz.

## Additional frontmatter
```yaml
target_role: ""          # e.g. "Enterprise Architect, 20+ years experience"
```

## Required section skeleton
1. H1 title + H2 "How to use this guide" explaining the answer-layer
   structure (see below) — every question bank in this repo should teach the
   same three-layer method so a reader who's used one can use any of them.
2. Sectioned by topic/theme (H2 per section), not one flat list — match the
   "SECTION 1: Strategic Architecture & AI Vision" pattern.
3. Each question, in order:
   - The question itself, in bold, phrased exactly as an interviewer would
     ask it (scenario-based, with realistic specifics — a company size, a
     constraint, a deadline — not abstract).
   - **Interviewer's Intent** — what's actually being tested.
   - **Thinking Approach** — how to structure a response before speaking.
   - **Model Answer** — a full senior-level response.

## Depth-of-research rubric
- Every question must be scenario-based with concrete specifics (numbers,
  constraints, named pressures), not generic ("Tell me about a time you led
  a project").
- All three layers (Intent / Approach / Model Answer) are required for every
  question — a question with just an answer isn't teaching the reasoning,
  which is the actual value of this format at senior level.
- Target 15-30 questions per bank, grouped into 4-6 thematic sections. Below
  that, it reads as thin; above that without more sections, it becomes
  unnavigable — split into a second bank by sub-specialty instead of growing
  one file indefinitely.

## Formatting notes
- Bold the question and the three layer-labels so a reader skimming the
  page can jump straight to whichever layer they need.
