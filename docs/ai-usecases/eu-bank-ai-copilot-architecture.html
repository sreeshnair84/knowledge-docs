Platform Overview & Design Constraints

This document describes the end-to-end architecture for an internal AI Copilot platform serving a regulated EU-based banking institution. The platform enables business teams to dynamically compose AI agent workflows backed by Model Context Protocol (MCP) tools, presented through a CopilotKit-compatible chat/copilot UI, with all orchestration handled by the Strands Agents SDK running on AWS AgentCore Runtime.

Design Constraint — AgentCore Gateway AgentCore Gateway is not an approved service in this environment. All external routing to AgentCore Runtime is performed exclusively through the BFF layer. MCP servers are deployed as AgentCore Runtime workloads. No direct client-to-AgentCore calls are permitted.

Key Design Principles

Zero Trust

Every service-to-service call is authenticated with short-lived credentials. No implicit trust based on network location.

Regulatory First

GDPR, DORA, EBA ICT Guidelines, NIS2, and MiCA (where applicable) are primary constraints, not afterthoughts.

Dynamic Composition

Teams register new MCP tools and CopilotKit UI elements via a Tool Registry without redeploying the core platform.

Audit Completeness

Every agent decision, tool invocation, and data access is immutably logged for supervisory review.

Data Residency

All processing, storage, and inference must remain within EU AWS regions (eu-west-1 / eu-central-1).

Human-in-Loop

High-risk operations (payments, risk overrides) require explicit human approval through CopilotKit interrupt flows.

Component Deep-Dive

3.1 CopilotKit Frontend

The React frontend integrates CopilotKit's CopilotChat, useCopilotAction, and CopilotKit provider. It communicates exclusively with the BFF; no direct calls to AgentCore or Bedrock are permitted from the browser.

Key Frontend Responsibilities

  • PKCE OAuth 2.0 flow with Entra ID (MSAL.js library)
  • Attach Authorization: Bearer <id_token> to all BFF requests
  • Dynamic UI element loading via Tool Registry manifest (Section 6)
  • Render CopilotKit generative UI components returned in agent SSE stream
  • Human-in-the-loop approval modals for gated operations
  • CSP enforcement: default-src 'self', no inline scripts

3.2 BFF — Backend For Frontend

The BFF is the single authoritative entry point. It is a hardened Node.js (or FastAPI) service running on EKS with an IRSA (IAM Roles for Service Accounts) role that allows it to invoke AgentCore Runtime endpoints.

3.3 Strands Agent on AgentCore Runtime

The Strands Agents SDK orchestrates multi-turn conversations and tool calls. Each agent worker container runs with a dedicated IAM Task Role scoped to minimum necessary permissions.

# strands_agent/main.py (simplified)
from strands import Agent, tool
from strands.models import BedrockModel
from mcp import MCPClient

async def build_agent(session_context: dict) -> Agent:
    # Discover tools from registry at runtime
    registry = await ToolRegistry.fetch(session_context["team_id"])
    mcp_tools = await MCPClient.list_tools(registry.mcp_endpoints)

    model = BedrockModel(
        model_id="anthropic.claude-3-5-sonnet-20241022-v2:0",
        region_name="eu-west-1",
        guardrails={"guardrailId": GUARDRAIL_ID}
    )

    return Agent(
        model=model,
        tools=mcp_tools,
        system_prompt=build_system_prompt(session_context),
        # Human-in-loop: pause before payment tools
        interrupt_before=["payment_execute", "limit_override"],
        callback_handler=AuditCallbackHandler(session_context["trace_id"])
    )

3.4 MCP Servers on AgentCore Runtime

Each domain team owns one or more MCP server containers deployed on AgentCore Runtime. The Tool Registry tracks available endpoints, capability manifests, and access policies.

Core Banking MCP

Account balance, transaction history (read-only). Calls internal Core Banking API via private link. All responses are filtered by customer ownership before returning.

Payment Rail MCP

SEPA / SWIFT payment initiation. Gated by human approval interrupt. Dual-control required above €10k. Idempotency key enforced.

Risk & Compliance MCP

Credit scoring lookup, regulatory limit checks, exposure queries. Returns structured compliance verdicts. Never returns raw PII.

KYC / AML MCP

Identity verification status, PEP screening, sanctions check. Results immutably logged. No PII written to conversation history.

Dynamic UI Loading in CopilotKit

// DynamicToolLoader.tsx
import { useCopilotAction } from "@copilotkit/react-core";
import { useToolRegistry } from "../hooks/useToolRegistry";

export function DynamicToolLoader() {
  const { tools } = useToolRegistry(); // fetches manifest from BFF on mount

  tools.forEach((tool) => {
    // Lazy-load UI component with SRI verification
    const UIComponent = React.lazy(() =>
      loadWithSRICheck(tool.ui.bundle_url, tool.ui.integrity)
    );

    // Register CopilotKit action dynamically
    useCopilotAction({                   // eslint-disable-line
      name: tool.copilotkit_actions[0].name,
      description: tool.copilotkit_actions[0].description,
      parameters: tool.copilotkit_actions[0].parameters_schema,
      render: ({ args, status }) => (
        <Suspense fallback={<Spinner />}>
          <UIComponent args={args} status={status} />
        </Suspense>
      ),
      handler: async (args) => {
        // BFF proxies to AgentCore; never direct
        return await bffClient.post("/copilot/tool-action", {
          tool_id: tool.tool_id,
          args,
        });
      },
    });
  });

  return null;
}
Security — SRI + CSP for Dynamic UI Every dynamically loaded UI bundle must pass Subresource Integrity (SRI) sha384 verification. The Content Security Policy includes require-sri-for script style. Any bundle whose hash does not match the registry manifest is rejected. Bundles are served from a bank-controlled CDN only.

8.3 EBA ICT Guidelines & PSD2

  • Strong Customer Authentication (SCA) — Entra ID Conditional Access enforces MFA for all sessions, satisfying PSD2 SCA for staff-facing tools.
  • Audit Trail — Immutable audit log (S3 Object Lock — WORM) with 7-year retention meets EBA record-keeping requirements.
  • Segregation of Duties — Maker/Checker enforced in Approval Queue; agent cannot self-approve gated operations.
  • Outsourcing Notification — AgentCore Runtime use documented as material ICT outsourcing; EBA notification template filed.
  • Business Continuity — Fallback mode: if AgentCore is unavailable, BFF returns graceful degradation UI; no silent failure.

8.4 AI Act (EU 2024/1689) Considerations

High-Risk AI System Classification AI systems used in banking for credit decisions, risk assessment, or customer-facing decisions may qualify as High-Risk under Annex III. This platform operates in an internal staff assistance mode; however, the architecture includes the following safeguards to maintain compliance readiness:
  • Human oversight mandatory for all consequential actions (payments, risk overrides)
  • Model transparency: agent responses include source citations and confidence indicators
